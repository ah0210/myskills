# Playwright 网页抓取 GUI 工具

一个使用 Playwright 实现的网页抓取工具，提供图形界面进行操作，支持分页抓取和内容选择功能。

## 功能特性

- **图形界面**：使用 tkinter 实现的直观 GUI 界面
- **Playwright 驱动**：使用 Playwright 进行网页抓取，支持动态渲染
- **分页抓取**：支持自动分页抓取，可设置最大页数
- **内容选择**：支持通过 CSS 选择器选择要抓取的内容
- **多种内容类型**：支持抓取链接、标题、段落、图片等内容
- **无头/有头模式**：可选择使用无头浏览器或有头浏览器
- **结果保存**：支持将抓取结果保存到文件
- **实时日志**：显示抓取过程的实时日志

## 系统要求

- Python 3.8 或更高版本
- Windows、Linux 或 macOS
- 网络连接

## 安装步骤

### Windows 系统

1. **进入项目目录**：
   ```powershell
   cd skills/crawler_gui
   ```

2. **运行设置脚本**：
   ```powershell
   .\setup.bat
   ```
   该脚本会：
   - 创建虚拟环境
   - 安装依赖包
   - 安装 Playwright 浏览器

3. **激活虚拟环境**：
   ```powershell
   .\.venv\Scripts\Activate.ps1
   ```

### Linux/Mac 系统

1. **进入项目目录**：
   ```bash
   cd skills/crawler_gui
   ```

2. **运行设置脚本**：
   ```bash
   chmod +x setup.sh
   ./setup.sh
   ```

3. **激活虚拟环境**：
   ```bash
   source .venv/bin/activate
   ```

## 快速开始

### 运行程序

```powershell
# 在激活虚拟环境的情况下
python main.py
```

### 使用步骤

1. **输入目标 URL**：在 "目标网站" 部分输入要抓取的网站 URL

2. **选择浏览器模式**：
   - 无头模式：后台运行，不显示浏览器窗口
   - 有头模式：显示浏览器窗口，便于调试

3. **设置分页抓取**（可选）：
   - 勾选 "启用分页抓取"
   - 输入下一页按钮的 CSS 选择器
   - 设置最大抓取页数

4. **选择内容类型**：
   - 选择要抓取的内容类型（链接、标题、段落、图片、自定义）
   - 输入或选择 CSS 选择器
   - 可使用预设按钮快速设置常用选择器

5. **开始抓取**：
   - 点击 "开始抓取" 按钮
   - 查看日志区域的抓取进度
   - 结果会显示在结果区域

6. **保存结果**：
   - 抓取完成后，点击 "保存结果" 按钮
   - 结果会保存到 `output` 目录

## CSS 选择器示例

### 常用选择器

- **链接**：`a`
- **标题**：`h1, h2, h3, h4, h5, h6`
- **段落**：`p`
- **图片**：`img`
- **所有文本**：`*`

### 分页按钮选择器示例

- `.next`：类名为 next 的元素
- `.pagination-next`：类名为 pagination-next 的元素
- `#next`：ID 为 next 的元素
- `a[rel="next"]`：rel 属性为 next 的链接

### 高级选择器

- `.article .title`：class 为 article 的元素下的 class 为 title 的元素
- `div > p`：div 元素的直接子 p 元素
- `ul li:first-child`：ul 元素的第一个 li 子元素

## 常见问题

### 1. 抓取失败

- **原因**：可能是网络问题、网站反爬、选择器错误等
- **解决方法**：
  - 检查网络连接
  - 尝试使用有头模式查看浏览器行为
  - 检查 CSS 选择器是否正确
  - 尝试增加抓取间隔（修改 crawler.py 中的 slow_mo 值）

### 2. 分页抓取不工作

- **原因**：下一页按钮选择器错误或网站分页机制不同
- **解决方法**：
  - 使用浏览器开发者工具查看下一页按钮的实际选择器
  - 尝试不同的选择器
  - 检查网站是否使用 JavaScript 加载下一页内容

### 3. 内容抓取不完整

- **原因**：内容可能需要滚动加载或延迟加载
- **解决方法**：
  - 在 crawler.py 中添加滚动逻辑
  - 增加等待时间
  - 检查选择器是否正确

### 4. Playwright 浏览器安装失败

- **原因**：网络问题或权限问题
- **解决方法**：
  - 确保网络连接正常
  - 以管理员权限运行设置脚本
  - 手动安装：`python -m playwright install`

## 性能优化

1. **使用无头模式**：在生产环境中使用无头模式，减少资源消耗
2. **合理设置最大页数**：根据需要设置合适的最大页数，避免过度抓取
3. **优化选择器**：使用更精确的 CSS 选择器，减少不必要的元素抓取
4. **增加等待时间**：对于动态加载的网站，适当增加等待时间

## 注意事项

1. **遵守 robots.txt**：尊重网站的 robots.txt 规则
2. **控制抓取频率**：不要过度抓取，避免对目标网站造成压力
3. **处理反爬**：某些网站可能有反爬机制，需要适当处理
4. **隐私保护**：不要抓取敏感信息
5. **法律合规**：确保抓取行为符合相关法律法规

## 故障排除

### 1. 虚拟环境激活失败

- **症状**：无法激活虚拟环境
- **解决方法**：
  - 检查 Python 版本是否 >= 3.8
  - 重新运行设置脚本
  - 手动创建虚拟环境

### 2. Playwright 浏览器启动失败

- **症状**：浏览器无法启动
- **解决方法**：
  - 确保 Playwright 浏览器已正确安装
  - 检查系统权限
  - 尝试使用不同的浏览器（修改 crawler.py 中的浏览器类型）

### 3. GUI 界面无响应

- **症状**：界面卡顿或无响应
- **解决方法**：
  - 检查抓取任务是否过于复杂
  - 减少最大抓取页数
  - 优化选择器
