# Rubrics Eval Skill

LLM 质量控制工具箱，基于标准化可量化的评测体系，告别"凭感觉"。

---

## 🎯 核心能力

| 命令 | 功能 |
|------|------|
| `review <文件/代码>` | 5 维度加权代码评审 |
| `eval model` | 11 维度标准化模型评测 |
| `prompt <文件/内容>` | Prompt 质检与自动优化 |

---

## 🎯 核心设计：官方五层 10 分制架构 v2.2

✅ v2.2 三大纠偏机制：难度系数+基准线归一化+高分压缩，彻底解决分数虚高

| 层级 | 满分 | 权重 | 说明 |
|------|------|------|------|
| 第一层：基础能力 | 2.0 分 | **20%** | 降低权重，解决拉不开差距 |
| 第二层：工具专项能力 | 3.5 分 | **35% 🔴 核心** | Agent 和普通 LLM 的分界线 |
| 第三层：场景化适配能力 | 2.0 分 | **20%** | 不同工作场景的适应性 |
| 第四层：性能&安全能力 | 1.5 分 | **15%** | 工业级生产环境要求 |
| 第五层：惩罚/奖励项 | 1.0 分 | **10% ✅ 新增** | 幻觉/错误扣分，零错误奖励 |
|--------|------|------|------|
| **总分** | 10.0 分 | **100%** | |

## 🚀 使用流程

### 1. 代码评审

```python
# 评审单个文件
exec("python3 skills/rubrics-eval/scripts/rubrics.py review path/to/code.py")
```

### ⚙️ 配置说明

首次使用前，请先修改 `config.json` 配置文件，将模型名称替换为您环境中的实际模型：

```json
{
  "被测模型映射": {
    "rose": "your-model-name",
    "elm": "your-main-model"
  },
  "打分模型": "your-judge-model",
  "默认被测模型": "elm",
  "评测设置": {
    "随机抽题数量": 5,  // 增量评测每次抽题数量
    "题目超时秒数": 120
  }
}
```

**输出：** 5 维度加权评分 + P0/P1/P2 分级修改建议

---

### 2. 模型评测（五层 10 分制）

```python
# ✅ 推荐：增量评测模式，随机抽取5道题（数量可配置），快速评估
exec("python3 scripts/rubrics.py eval model --random5 --manual")

# 盲测模式：出题 → 答题 → 第三方打分，全程匿名
exec("python3 scripts/rubrics.py eval model --blind")
```

**增量评测说明：**
- 每次从全部题库随机抽取 N 道题（默认5道，可在 config.json 配置）
- 自动按层级均衡抽样，确保覆盖基础、工具、场景、安全各维度
- 多次增量评测结果可累加，逐步完善模型画像
- 适合快速迭代、持续集成的评测场景

| 阶段 | 说明 |
|------|------|
| 阶段 1 | 运行 3 大类 10 道压力测试题 |
| 阶段 2 | 所有答案匿名化，打乱顺序 |
| 阶段 3 | 输出打分模板，人工或第三方模型揭盲打分 |

**✅ 坚决杜绝：自己答题自己打分！**

---

### 3. Prompt 质检

```python
# 给 Prompt 打分并自动注入 6 项质检
exec("python3 scripts/rubrics.py prompt path/to/prompt.md")
```
