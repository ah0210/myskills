## 🚫 问题1：API插件未启用导致自动评测失败
### 问题现象
运行自动评测脚本时，出现以下错误：
```
Failed to start CLI: Error: The `openclaw api` command is unavailable because `plugins.allow` excludes "api".
Add "api" to `plugins.allow` if you want that bundled plugin CLI surface.
```
### 根本原因
OpenClaw默认没有启用API插件，脚本无法通过API调用其他模型。
### 解决方案
#### 方案A：启用API插件（推荐，实现全自动评测）
修改OpenClaw配置文件，在`plugins.allow`数组中添加：
```json
"plugins.allow": ["api", "sessions-spawn", "sessions-history"]
```
重启OpenClaw后即可使用全自动评测功能。
#### 方案B：使用手动模式（无需修改配置）
运行脚本时添加`--manual`参数，进入手动交互模式，人工输入每个模型的回答：
```bash
python3 rubrics.py eval model rose --random5 --manual
```
#### 方案C：多Agent切换回答（半自动化）
直接输出题目，切换到不同模型回答后统一打分，无需修改脚本和配置。

## 🚫 问题2：sessions-spawn插件未启用导致子会话调用失败
### 问题现象
使用子会话调用其他模型时，出现以下错误：
```
The `openclaw sessions-spawn` command is unavailable because `plugins.allow` excludes "sessions-spawn".
Add "sessions-spawn" to `plugins.allow` if you want that bundled plugin CLI surface.
```
### 根本原因
sessions-spawn插件未启用，无法创建子会话调用其他模型。
### 解决方案
1. 同问题1的方案A，启用`sessions-spawn`和`sessions-history`插件
2. 或使用手动模式/多Agent切换回答的方式进行评测

---

## 🚫 问题3：非交互式环境下input()函数报错
### 问题现象
在CLI环境运行手动模式脚本时，出现以下错误：
```
EOFError: EOF when reading a line
```
### 根本原因
非交互式环境没有标准输入流，`input()`函数无法获取用户输入。
### 解决方案
1. 在交互式终端环境中运行脚本
2. 或直接输出题目，由Agent在会话中直接回答，无需通过脚本输入
3. 或修改脚本，从命令行参数或文件中读取回答内容

---

## 🚫 问题4：批量评测空跑导致所有模型得分为0
### 问题现象
批量评测运行完成后，所有模型得分都是0分，没有实际答题内容。
### 根本原因
插件未启用，脚本无法调用模型，所有题目都被跳过。
### 解决方案
1. 启用必要的插件后重新运行
2. 或使用手动模式逐个模型进行评测
3. 空跑结果无效，需要重新评测

## ✅ 最佳实践建议
### 1. 环境准备
- 首次使用前先检查插件是否启用：`openclaw config get plugins.allow`
- 确保需要评测的模型都已在Agent配置中正确配置
- 建议先测试单模型评测，确认可用后再运行批量评测

### 2. 评测方式选择
| 场景 | 推荐方式 | 优点 | 缺点 |
|------|----------|------|------|
| 环境配置完整，插件齐全 | 全自动模式（默认） | 全程无需人工干预，效率高 | 需要配置插件 |
| 无法修改配置，需要快速评测 | 多Agent切换回答 | 无需任何配置，结果准确 | 需要人工切换模型 |
| 评测模型少，需求简单 | 手动模式 | 灵活可控 | 效率较低 |

### 3. 评分注意事项
- 严格按照评测标准打分，避免主观偏差
- 同一套题目所有模型使用相同的评分尺度
- 建议由独立的第三方模型进行打分，避免自评测的公平性问题
- 应用高分压缩机制，避免分数虚高，拉开模型差距

## 📋 配置文件常见问题

### 🚫 问题5：模型名称配置错误
**问题现象：**
```
模型调用失败：No such model: ark-test/anonymous/rose
```
**根本原因：**
配置文件中的模型名称与实际环境不匹配。

**解决方案：**
1. 打开 `config.json` 文件
2. 检查 `被测模型映射` 和 `打分模型` 中的模型名称
3. 替换为您 OpenClaw 环境中实际存在的模型名称
4. 检查 `默认被测模型` 也要对应修改

### 🚫 问题6：如何添加新的评测模型
**解决方案：**
直接在 `config.json` 的 `被测模型映射` 中添加新的键值对即可：
```json
"被测模型映射": {
  "my-model": "your-org/my-model-name",
  "another": "another-model"
}
```
命令行中直接使用短名调用：
```bash
python3 scripts/rubrics.py eval model my-model --random5
```

### 🚫 问题7：如何修改随机抽题数量
**解决方案：**
修改 `config.json` 中的 `随机抽题数量` 参数：
```json
"评测设置": {
  "随机抽题数量": 10,  // 改为您想要的数量
  "题目超时秒数": 120
}
```
支持 1-19 道题的任意数量。
