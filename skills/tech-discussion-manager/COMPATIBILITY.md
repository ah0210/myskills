### 依赖检查

✅ **零运行时依赖**
- 不需要安装任何软件包
- 所有安装脚本使用基础 Shell 命令
- 不需要 Python、Node.js 等运行环境
- 不需要数据库或其他服务

## 🔧 Shell 命令兼容性验证

所有安装脚本使用的命令都符合标准规范，跨平台兼容：

| 命令 | 兼容性 | 说明 |
|------|--------|------|
| `mkdir -p` | ✅ 全平台 | POSIX 标准，所有系统支持 |
| `cp -n` | ✅ 全平台 | 不覆盖已存在文件，避免覆盖用户自定义 |
| `2>/dev/null \|\| true` | ✅ 全平台 | 静默忽略错误，跨平台兼容 |
| `"$VAR"` 引用 | ✅ 全平台 | 正确处理路径中的空格 |

## 🌍 编码兼容性

✅ 所有文件使用 UTF-8 无 BOM 编码，支持中英文等多语言
✅ 换行符使用 LF 格式，兼容 Linux/macOS，Windows 自动转换

## 📁 路径兼容性

✅ 使用 `$OPENCLAW_WORKSPACE` 环境变量，自动适配不同工作空间路径
✅ 所有路径使用正斜杠，符合跨平台规范
✅ 正确处理路径中的特殊字符和空格（通过引号包裹）

## 🤝 多智能体兼容性

✅ 完全兼容多智能体环境
- 统一存储结构，多个智能体可以同时操作
- 文件名使用日期+主题格式，自动避免冲突
- 不修改用户工作区配置文件，无侵入性
- 不需要锁机制，纯文件操作无并发问题

## 🚀 跨工作空间兼容性

✅ 支持多个工作空间独立使用
- 每个工作空间有独立的讨论数据
- 技能本身和数据分离，一份技能安装支持多个工作空间
- 工作空间之间完全隔离，互不影响

## ⚠️ 已知限制

1. 工作空间路径包含特殊字符（如换行符）可能导致异常（极罕见场景）

- Windows 11 (PowerShell / WSL2)
