# tech-discussion-manager 技术讨论管理技能

[![OpenClaw Skill](https://img.shields.io/badge/OpenClaw-Skill-blue)](https://openclaw.ai)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

标准化技术讨论和创意沉淀技能，自动创建规范的目录结构和记录模板，让智能体帮你自动管理技术知识。

## ✨ 特性

- 📝 **标准化模板**：统一的讨论记录格式，包含背景、要点、决策、待办等
- 📂 **自动目录创建**：每个讨论自动生成对应代码输出目录
- 🔄 **自动索引维护**：自动更新讨论索引，方便检索
- 📦 **归档机制**：历史讨论一键归档，保持目录整洁
- 🤝 **多智能体协同**：统一存储结构，支持多智能体协同工作
- 🚀 **零依赖**：纯规则驱动，不需要运行任何服务或脚本
- 🔒 **安全设计**：不修改用户工作区配置文件，尊重用户选择权

## 🚀 安装

### 通过 SkillHub 安装（推荐）
```bash
skillhub install tech-discussion-manager
```

### 手动安装
1. 下载技能到 OpenClaw 技能目录
2. 安装会自动创建目录结构和模板文件

## 🎯 使用方式

### 触发关键词
当对话中包含以下词汇时，智能体可以触发技能：
- "讨论一下"、"技术评审"、"方案设计"
- "需求评审"、"架构设计"、"brainstorm"
- "头脑风暴"、"记录一下"、"存档"

**注意：** 也可以直接在当前对话中记录讨论，不强制使用技能。

### 使用示例
```
用户：我们来讨论一下GEO工具的架构设计
智能体：✅ 已为你创建技术讨论记录：
📝 讨论文件：memory/tech-discussions/2026-04-02-geo工具架构设计.md
📂 代码目录：code-output/geo工具架构设计/
我们可以开始讨论，我会实时记录要点。
```

## 📁 存储结构

所有数据都保存在工作空间，完全可控：

```
your-workspace/
├── memory/
│   └── tech-discussions/
│       ├── archive/                # 已归档的历史讨论
│       ├── TEMPLATE.md             # 讨论记录模板
│       ├── index.md                # 讨论索引目录
│       └── YYYY-MM-DD-主题.md       # 具体讨论记录
├── code-output/
│   └── 主题名称/                    # 每个讨论对应的代码输出目录
└── docs/
    ├── decisions/                  # 沉淀的正式技术决策
    └── architecture/               # 架构设计文档
```

## 📝 讨论记录模板

```markdown
# [主题] 技术方案讨论
时间：2026-04-02 15:30
参与方：用户、智能体
状态：进行中 / 已完成
标签：[标签1] [标签2]

## 讨论背景
[讨论背景描述]

## 讨论要点
- [ ] 要点1：xxx
- [ ] 要点2：xxx

## 决策结论
- 结论1：xxx，原因：xxx
- 结论2：xxx，原因：xxx

## 待办事项
- [ ] 任务1：xxx，负责人：xxx，截止时间：xxx

## 相关产出
- 代码输出：`code-output/[主题]/`
- 文档：`docs/`
- 参考链接：
```

## 🤝 多智能体协同

多个智能体可以同时使用本技能：

- 统一存储结构，所有讨论都保存在工作空间
- 自动避免文件名冲突（日期+主题格式）
- 支持协同编辑同一份讨论记录
- 状态实时同步
